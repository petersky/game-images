Game asset workflows (guided steps)

Click a step to open the right tool. Run each step when ready; the active image updates as you go.

Tileable texture

  1. Create a seamless-friendly albedo
  2. Tile → offset seam to center
  3. Manipulate → mask & fix center blend
  4. Maps → normal / roughness

Wider environment / backdrop

  1. Zoom out for more scene around the subject
  2. Extend one side if you need asymmetric fill
  3. Tile mirror to check continuity (optional)

Sprite / icon focus

  1. Zoom in on the subject
  2. Manipulate small fixes with a tight mask
  3. Adjust size for target resolution

Shift then fill (panorama strip)

  1. Prepare → shift to expose blank edge
  2. Extend → fill the blank region
  3. Repeat or Zoom out for uniform growth

Adjust

Tile / seamless

Offset swaps halves so seams move to the center; refine edges with Manipulate. Mirror modes help check tiling.

Texture maps

Prepare — shift

Move the image in a direction; the opposite side becomes black. Then use Extend to fill that area (e.g. Shift west → blank on right → Extend east). Shift is applied in memory until you run another operation.

Zoom

Zoom out adds canvas around the image (AI outpaint). Zoom in crops a tighter frame; optionally enhance with AI.

Outpaint: describe what should appear in the new border areas. Try 1.25×–1.5× per step for best coherence.

Dashed box in Preview shows the crop region.

Extend (outpaint)

Adds pixels on chosen sides. The API gets a larger canvas + mask (transparent = new areas). Prompt what should appear in the new border — not only “extend”. Use 128–256 px per step for cleaner results.

Manipulate (inpaint)

Describe the whole final image, including what must stay the same. A mask is sent with Run (transparent = edit). Without a mask the model guesses the region — use a mask for game-art fixes.

Transparent = where to apply changes; opaque = keep unchanged. Preview tint shows edit areas only — the API receives the real mask PNG.

No mask set — Run will edit the whole image from the prompt only.

Settings

API keys

Keys are saved locally in your Game Images data folder. Environment variables override stored values when set.

OpenAI

Advanced: paste token manually

Create, Extend, and Manipulate require a platform API key (sk-…), not ChatGPT sign-in. Sign-in is stored for future use but cannot call the Images API today. Choose API key mode when both are configured. Sign-in uses port 1455 (see README if the callback fails).
